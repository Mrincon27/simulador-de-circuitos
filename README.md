### **Aspectos de POO**
El c√≥digo utiliza varios conceptos de Programaci√≥n Orientada a Objetos (POO) para estructurar la l√≥gica del simulador:

1. **Herencia:**
   - Las clases `FuenteDC`, `Resistencia`, `Capacitor` e `Inductor` heredan de la clase base `Componente`. Esto permite reutilizar c√≥digo y definir comportamientos comunes para todos los componentes.
   - Las clases `CircuitoRC`, `CircuitoRL`, `CircuitoRLC`, `CircuitoRC_Paralelo`, `CircuitoRL_Paralelo` y `CircuitoRLC_Paralelo` heredan de la clase base `Circuito`. Esto permite definir comportamientos espec√≠ficos para cada tipo de circuito.

2. **Encapsulamiento:**
   - Los atributos de las clases est√°n encapsulados mediante el uso de propiedades (`@property`). Esto permite controlar el acceso y la modificaci√≥n de los valores de los componentes, asegurando que no se asignen valores inv√°lidos.

3. **Polimorfismo:**
   - Cada tipo de circuito implementa sus propios m√©todos para calcular voltajes y corrientes, pero todos comparten una interfaz com√∫n gracias a la herencia. Esto permite que la simulaci√≥n se realice de manera uniforme, independientemente del tipo de circuito.

4. **Abstracci√≥n:**
   - La clase `Componente` es una abstracci√≥n que define los atributos y m√©todos comunes a todos los componentes del circuito. Las clases derivadas implementan detalles espec√≠ficos.

### **Casos de Uso**
1. **Simulaci√≥n de un circuito RC en serie:**
   - El usuario selecciona el circuito RC en el men√∫ principal.
   - Ingresa los valores de la resistencia, el capacitor y la fuente de voltaje.
   - El simulador calcula y muestra las gr√°ficas de voltaje en la resistencia, voltaje en el capacitor y corriente en el circuito.

2. **Simulaci√≥n de un circuito RL en paralelo:**
   - El usuario selecciona el circuito RL en el men√∫ principal.

   - El simulador calcula y muestra las gr√°ficas de voltaje en el circuito, corriente en la resistencia, corriente en el inductor y corriente total.

3. **Guardar y cargar configuraciones:**
   - El usuario puede guardar los valores de los componentes en un archivo JSON.
   - Posteriormente, puede cargar estos valores para realizar una simulaci√≥n sin necesidad de ingresar los datos nuevamente.

4. **Cambiar entre serie y paralelo:**
   - El usuario puede cambiar la configuraci√≥n del circuito entre serie y paralelo para observar c√≥mo afecta el comportamiento del circuito.
